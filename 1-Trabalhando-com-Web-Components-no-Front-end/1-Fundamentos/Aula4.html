Entenda o DOM

É explicado como navegador interari com seu código

  1. Quando digita os códigos no arquivo físico (HTML)

  2. Ele passa pelo navegador (browser), faz um processo chamado de parsed.

  3. Renderiza tudo em um modelo de arvore, chamado de D.O.M (Document Object Model)

O que acontece por baixo dos panos enquanto renderiza o DOM.

A relação entre os elementos, o elemento que ta mais abaixo é filho do elemento acima que é o pai, respeitando a hierarquia. Na imagem abaixo é um exemplo da arvore do código ao lado.
